Core AI Concepts

Foundation model

Broadly trained reusable model

Foundation model → many downstream tasks
Do not treat every foundation model as an LLM.

LLM

Language-focused foundation model

LLM ⊂ foundation models
Language is the focus, not the full category.

Generative vs discriminative

Create versus classify/predict

generative → produce
discriminative → decide
Many applications combine both.

Prompting and Templates

Zero-shot / few-shot

Control with no examples or several examples

zero-shot: instruction
few-shot: examples + new input
Bad examples teach bad patterns.

PromptTemplate

Reusable formatted text

PromptTemplate.from_template("Explain {topic}.")
prompt.invoke formats; it does not call the model.

ChatPromptTemplate

Reusable role-based messages

ChatPromptTemplate.from_messages([...])
Preserve system and human roles.

LangChain and Output

LCEL

Compose runnable steps

chain = prompt | model | parser
The pipe means sequence, not parallel.

RunnableParallel

Independent branches from one input

RunnableParallel(a=..., b=...)
Branch names become result keys.

Structured output

Validated application fields

model.with_structured_output(MySchema)
Know whether the result is dict or Pydantic.

Watson, RAG, Agents

Watson Route A

IBM generate — low level

from watson_helper import model
model.generate("...")
Not Runnable — cannot sit in LCEL pipe.

Watson Route B

LangChain LLM — invoke and pipes

from watson_llm import make_watsonx_llm
llm = make_watsonx_llm()
helper vs llm — similar names, different route.

RAG spine

Load → split → embed → Chroma → QA

RetrievalQA.from_chain_type(llm, retriever, chain_type="stuff")
Not the same as conversation memory (Lab 33).

Embeddings

Text → vectors for search

from watson_llm import make_watsonx_embeddings
emb.embed_documents(texts)
Not llm.invoke — different model job.

Memory buffer

Persist chat turns on chain

ConversationBufferMemory()
ConversationChain(llm=..., memory=memory)
LLM is stateless without memory or manual history.

Agent vs chain

Fixed pipe vs tool loop

chain = prompt | llm | parser
AgentExecutor(agent=..., tools=...)
Use chain when steps are known; agent when tool choice varies.

Application Delivery

MessagesPlaceholder

Insert prior messages

MessagesPlaceholder(variable_name="history")
Supply a list of message objects.

Model selection

Compare candidates using task evidence

baseline → compare → monitor
Do not choose by size or brand alone.

Flask route

Expose the chain through HTTP

@app.route("/explain", methods=["POST"])
Validate input and return explicit status codes.

Course Flow

Foundation model → Prompt → LangChain workflow → Structured output → Flask application
