1. Big Picture

This module moves from model capability to application control. Foundation models provide broad capability, LLMs focus that capability on language, prompt engineering directs the model at runtime, and templates make those directions reusable.

2. Generative AI and Foundation Models

Discriminative models learn boundaries for classification or prediction. Generative models learn patterns that support new text, images, audio, or code. A foundation model is trained broadly and adapted through prompting, retrieval, tools, or further training. An LLM is a language-focused foundation model.

task = "Summarize this customer email in three bullets."
What to notice: The model provides capability; the application still supplies task, context, constraints, and output shape.

3. Prompt Engineering and In-Context Learning

Zero-shot uses no examples, one-shot uses one, and few-shot uses several. Step-by-step prompting helps multi-stage problems. Self-consistency compares several candidate solutions and selects the strongest agreement.

prompt = """Classify the request.

Text: I was charged twice.
Category: billing

Text: {request}
Category:"""
Common trap: More examples are not automatically better; weak examples can teach the wrong pattern.

4. Reusable Prompt Templates

PromptTemplate produces formatted text. ChatPromptTemplate produces role-based messages. prompt.invoke(...) fills variables and prepares model input; it does not call the model.

Technique How it works Best use
Zero-shot No demonstrations Clear familiar tasks
Few-shot Several demonstrations Stable labels, tone, or format

5. Worked Example

The smallest proof uses one stable template with several runtime variables.

from langchain_core.prompts import PromptTemplate

prompt = PromptTemplate.from_template(
    "Explain {topic} to a {audience} using {tone} language."
)

formatted = prompt.invoke({
    "topic": "vector embeddings",
    "audience": "beginner",
    "tone": "friendly",
})
print(formatted.to_string())
Result: The tested output was “Explain vector embeddings to a beginner using friendly language.” No LLM call occurred.

6. Validation and Troubleshooting

  • Match every placeholder to an invocation key.
  • Inspect the formatted prompt before adding a model.
  • Use ChatPromptTemplate when roles must stay distinct.
  • Choose prompting techniques based on task complexity and examples available.
Red flag: Treating every foundation model as an LLM or assuming prompt.invoke(...) called the model.

7. Common Mistakes

Foundation model equals LLM An LLM is one language-focused type of foundation model.
Few-shot means fine-tuning Few-shot examples live in the runtime prompt; fine-tuning changes model behavior through training.
Prompt invocation means generation prompt.invoke prepares; model.invoke generates; chain.invoke runs all connected steps.

8. Interview Translation

What is a foundation model? A broadly trained model that supports many downstream tasks through prompting or adaptation.
What is prompt engineering? Designing instructions, context, examples, constraints, and output formats to guide a model.
Why use prompt templates? They improve reuse, consistency, testing, and maintainability.
