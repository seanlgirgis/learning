## Pages 5–8 — Sum calculator (`gradio_demo.py`)

### Concept

`gr.Number()` — numeric **input or output** field.

### Code (course)

```python
import gradio as gr

def add_numbers(Num1, Num2):
    return Num1 + Num2

demo = gr.Interface(
    fn=add_numbers,
    inputs=[gr.Number(), gr.Number()],
    outputs=gr.Number(),
)
demo.launch(server_name="127.0.0.1", server_port=7860)
```

### Run / view

- Terminal: `python3.11 gradio_demo.py`
- URL: `http://127.0.0.1:7860`
- Public link hint: `share=True` in `launch()`
- SN Labs: “Web Application” button → port **7860**
- Stop: **Ctrl+C**

## Pages 10–13 — Common inputs and outputs

### Inputs (know definitions verbatim for quiz)

| Component | Definition |
|-----------|------------|
| **Checkbox** | True or False **only** |
| **CheckboxGroup** | **Multiple** values from predefined list |
| **Dropdown** | **One** value by default; `multiselect=True` → one or more |
| **File** | User uploads file |
| **Image** | Select / upload image |
| **Radio** | Forces **one** value |
| **Slider** | Value between min and max; `value`=default, `step`=increment; **all int min/max/value/step → integer values** |
| **Textbox** | Expandable text |

### Outputs

| Component | When |
|-----------|------|
| `gr.Textbox()` or `"text"` | Most LLM apps |
| `gr.Label` | Classification — class + probability; `num_top_classes=N` limits rows |

| 3 | `CheckboxGroup([countries])` | countries (list) |
| 4 | `Radio([places])` | place |

## Pages 14–17 — Terminal LLM (`simple_llm.py`)

### watsonx.ai context

- Platform: Granite, Llama, Mixtral, …
- Use cases listed: Q&A, summarization, classification, generation, extraction, **RAG**, code generation
- Granite 3.3 8B adds: reasoning & planning, fill-in-the-middle

### Course code pattern

```python
from langchain_ibm import WatsonxLLM
from ibm_watsonx_ai.metanames import GenTextParamsMetaNames as GenParams

parameters = {
    GenParams.MAX_NEW_TOKENS: 256,
    GenParams.TEMPERATURE: 0.5,
}
watsonx_llm = WatsonxLLM(
    model_id='meta-llama/llama-3-2-11b-vision-instruct',  # or Mixtral line
    url="https://us-south.ml.cloud.ibm.com",
    project_id="skills-network",
    params=parameters,
)
query = input("Please enter your query: ")
print(watsonx_llm.invoke(query))
```

**Key facts:**

- `project_id = "skills-network"` — free in SN Labs only
- Switch model = change `model_id` comment block
- Test query example: “How to be a good data scientist?”

**Local:** You use `make_llm()` via `watson_llm.py` — same invoke pattern, different credentials.

## Pages 18–22 — Gradio + LLM (`llm_chat.py`)

### Three integration pieces

1. Initialize model **once** (module level)
2. `generate_response(prompt_txt)` → `watsonx_llm.invoke(prompt_txt)`
3. `gr.Interface` with Textbox in/out

```python
chat_application = gr.Interface(
    fn=generate_response,
    allow_flagging="never",
    inputs=gr.Textbox(label="Input", lines=2, placeholder="Type your question here..."),
    outputs=gr.Textbox(label="Output"),
    title="Watsonx.ai Chatbot",
    description="Ask any question and the chatbot will try to answer.",
)
chat_application.launch(server_name="127.0.0.1", server_port=7860)
```

**Local mirror:** `15_m2_gradio_llm_chat.py`

### Exercise (page 21–22) — incomplete responses

**Fix:** Increase `GenParams.MAX_NEW_TOKENS` (e.g. 512 or 1024).
