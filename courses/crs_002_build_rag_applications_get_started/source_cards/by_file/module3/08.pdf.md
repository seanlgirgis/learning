# File 8 — `08.pdf` (cheat sheet: LlamaIndex)

Line-by-line from one-page cheat sheet.

---

## Opening

LlamaIndex = framework for **LLM context augmentation** (like LangChain).

Use cases: chatbots, **RAG**, AI assistants.

---

## Document class

- Wraps **entire** document from store
- Stores: **text**, optional **embeddings**, **metadata** (created date, directory), **relationships** to other docs

**Loader:** `SimpleDirectoryReader` — files or whole directories. More loaders on **LlamaHub**.

---

## Nodes

- Documents chunked into **Nodes**
- Node structure **similar to Document** (metadata, relationships, embeddings)

---

## Splitters

- **SentenceSplitter:** recursive split on character list; each chunk ≤ max **token** length
- **LangChainNodeParser:** wrap any LangChain splitter

---

## Embedding + storage (one step)

**`VectorStoreIndex`**
- Default: in-memory
- Can wrap **Chroma, FAISS, Milvus**

---

## Retrieval

- From `VectorStoreIndex`: `as_retriever()`
- Same embedding model for nodes and user prompt

---

## Augment + LLM (one step)

**Response synthesizer:** input = user prompt + retrieved nodes → output = LLM response

**Query engine:** `as_query_engine()` — input = user prompt only → output = LLM response (retrieval internal)

---

## Prompt templates

Predefined text with placeholders for **user prompt** and **retrieved chunks**.

Lab placeholders: `{context_str}`, `{query_str}` (`05.pdf` config).

---

## LlamaIndex vs LangChain (cheat sheet closing)

| LlamaIndex | LangChain |
|------------|-----------|
| Simplicity, native tools | More integrations, modular |
| Faster dev for RAG index/query | Easier to customize each chain piece |
