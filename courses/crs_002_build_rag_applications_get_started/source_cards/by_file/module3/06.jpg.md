## Q1 — Document loading: LangChain vs LlamaIndex

**Pick:** LlamaIndex's `SimpleDirectoryReader` can natively handle **more file formats** than LangChain's DirectoryLoader.

| Option | Verdict |
|--------|---------|
| LangChain relies on external integrations vs LlamaIndex native | Partial — course pushes LlamaIndex simplicity |
| DirectoryLoader uses SimpleDirectoryReader under the hood | **False** |
| **SimpleDirectoryReader more native formats** | **✓ Course answer** |
| LlamaIndex requires extensive config per type | **False** |

---

## Q2 — Embedding and storing vectors

**Pick:** LlamaIndex uses a **single command** to generate and store vectors; LangChain does these **separately**.

Evidence: `VectorStoreIndex(nodes)` embeds + stores in one step (`02.md`, `08.pdf`).

---

## Q3 — Alex: directory + subdirs, CSV and JSON only

**Pick:** `SimpleDirectoryReader` with **`recursive=True`** and **`required_exts`**.

From `01.md`:

```python
SimpleDirectoryReader(path, recursive=True, required_exts=[".csv", ".json"])
```

| Wrong option | Why |
|--------------|-----|
| DirectoryLoader + file_types | LangChain name, not LlamaIndex API |
| input_files list of paths | Specific files, not “whole tree filtered by ext” |
| SimpleDirectoryReader no params | Loads folder defaults, not filtered recursive |

## Q4 — Role of `SentenceSplitter`

**Pick:** **Recursively splits** long text on characters like **newline** and **period**.

Not: embeddings, semantic similarity, or directory loading.

---

## Q5 — Customize prompt template — challenge?

**Pick:** Prompt augmentation is **combined with other steps**, which can complicate customization.

LlamaIndex **does** support templates (`text_qa_template`, `{context_str}`, `{query_str}` in `05.pdf` config) — but they're wired through query engine / synthesizer, not a lone `PromptTemplate` in a chain.
