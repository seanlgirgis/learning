## Section 1 — What is LlamaIndex?

- **Framework** for **LLM-powered context augmentation**
- **Context augmentation** = make *your* data available to an LLM so answers are **grounded** in that data (not only training memory)

### Typical use cases (video lists four)

| Use case | What it means |
|----------|----------------|
| **Q&A with RAG** | Primary focus of this video |
| **Chatbots** | RAG + multi-turn follow-ups / clarification |
| **Document understanding** | Read natural language, pull names, dates, figures |
| **Data extraction** | Structured or unstructured → semantic fields |

---

## Section 2 — LlamaIndex in a RAG pipeline (full chain)

```text
1. LOAD + CHUNK source documents
2. EMBED each chunk (one vector per chunk)
3. STORE vectors in a vector store (vector DB)
4. USER PROMPT → embed prompt with SAME embedding model
5. RETRIEVE similar chunks from vector store
6. AUGMENT prompt (user question + retrieved text)
7. LLM → context-aware answer
```

**Align with M1:** Same RAG story; LlamaIndex gives **classes** for each step instead of only LangChain chains.

## Section 3 — Step 1: Load into `Document`

### Supported formats (video list)

- `.txt`, PDF, Markdown, CSV, JSON, HTML
- Plus **connectors** (databases, cloud storage) beyond local files

### `Document` class — five parts (inspect with `.dict()`)

| Field | Purpose |
|-------|---------|
| **id** | Unique document ID |
| **embedding** | Placeholder if you embed whole doc (usually embed **nodes** instead) |
| **metadata** | Dict — origin, creation date, folder, etc. |
| **relationships** | Dict — links to other docs/items |
| **text** | Actual body |

**Minimal example (video):**

```python
from llama_index.core import Document

doc = Document(text="Hello LlamaIndex")
doc.dict()  # inspect id, metadata, relationships, text
```

**Idea:** `Document` = **whole file** container before chunking.

## Section 4 — `SimpleDirectoryReader` (local loading)

**Import:** `from llama_index.core import SimpleDirectoryReader`

| Need | How |
|------|-----|
| All files in one folder | `SimpleDirectoryReader("path/to/dir").load_data()` |
| Folder + subfolders | `SimpleDirectoryReader("path", recursive=True).load_data()` |
| Specific files only | `SimpleDirectoryReader(input_files=["a.txt", "b.pdf"]).load_data()` |
| Only certain extensions | `SimpleDirectoryReader(required_exts=[".csv", ".json"]).load_data()` |

**Returns:** `list` of **`Document`** objects.

**Quiz tie-in (06.jpg Q3):** recursive + `required_exts` for CSV/JSON under subdirs.

## Section 5 — Chunking: `Document` → `Node`

- A **Node** = one **text chunk** (after splitting)
- **Why chunk?** Smaller segments → more precise embeddings → more relevant retrieval

### `SentenceSplitter`

**Import:** `from llama_index.core.node_parser import SentenceSplitter`

```python
splitter = SentenceSplitter(chunk_size=..., chunk_overlap=...)
nodes = splitter.get_nodes_from_documents(documents)
```

| Parameter | Meaning |
|-----------|---------|
| **chunk_size** | Max chunk size in **tokens** |
| **chunk_overlap** | Max tokens shared between adjacent chunks |

**How it splits:** Recursive — breaks on characters like **newlines** and **periods** until chunks fit size limit.

**Returns:** `list` of **text node** instances (structure similar to `Document`).

**Quiz tie-in (06.jpg Q4):** SentenceSplitter = recursive split on newline/period — **not** semantic similarity, **not** loading, **not** embedding.

### Other splitters (video mentions)

| Splitter | Behavior |
|----------|----------|
| **Semantic splitter** | Split when sentence similarity drops below threshold |
| **LangChain wrapper** | Use any LangChain splitter inside LlamaIndex |
