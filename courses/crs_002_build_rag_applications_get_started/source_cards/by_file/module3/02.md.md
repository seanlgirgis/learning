## Section 2 — `VectorStoreIndex` (embed + store)

**Class:** `VectorStoreIndex` — generates embeddings **and** stores them.

### Simple path (in-memory)

```python
index = VectorStoreIndex(nodes)
```

- Default embedding model
- Vectors stored **in memory**
- Only parameter: `nodes`

### Advanced path (custom embed + persistent store)

1. Import vector DB (e.g. **Chroma**) + embedding model (e.g. HuggingFace)
2. Define `embed_model`
3. Build `storage_context` pointing at Chroma
4. Call:

```python
index = VectorStoreIndex(
    nodes=nodes,
    embed_model=embed_model,
    storage_context=storage_context,
)
```

**Key idea (quiz Q2):** LlamaIndex **one class / one step** embeds and stores. LangChain often: embed documents, then separately add to Chroma.

---

## Section 3 — Retrieval: `as_retriever()`

```python
retriever = index.as_retriever()
results = retriever.retrieve("user prompt here")
```

- User prompt is **embedded with the same model** used for nodes (when using index retriever)
- Returns **ranked list** — most similar nodes first

### Limit how many chunks: `similarity_top_k`

```python
retriever = index.as_retriever(similarity_top_k=5)
```

Retrieves **k** most similar nodes (lab uses `SIMILARITY_TOP_K` in config).

---

## Section 4 — Response synthesizer (middle layer)

If you **already have** retrieved nodes:

```python
response = synthesizer.synthesize(query, nodes)
```

**Does in background:**
- Embed user prompt (if needed)
- **Prompt augmentation** (stuff context + question)
- Call LLM
- Return response

You do **not** manually stitch prompt strings for the basic path.

---

## Section 5 — Query engine (simplest app path)

```python
query_engine = index.as_query_engine()
response = query_engine.query("user question")
```

**Combines in one call:**
1. Prompt embedding
2. Retrieval
3. Prompt augmentation
4. LLM query
5. Response generation

**Lab (`05.pdf` Part 6):**

```python
query_engine = index.as_query_engine(
    streaming=False,
    similarity_top_k=config.SIMILARITY_TOP_K,
    llm=watsonx_llm,
    text_qa_template=facts_prompt,
)
response = query_engine.query(query)
return response.response
```

## Section 6 — Customization (video tips)

| Knob | What |
|------|------|
| Change **LLM** | Pass `llm=` to `as_query_engine` |
| **Prompt template** | `text_qa_template=PromptTemplate(...)` with `{context_str}`, `{query_str}` |
| **Retriever** | `as_retriever(similarity_top_k=...)` or custom retriever |

**Quiz tie-in (06.jpg Q5):** Augmentation is **bundled** with retrieval/LLM in query engine → customizing templates means learning where `text_qa_template` plugs in — not that LlamaIndex forbids templates.
