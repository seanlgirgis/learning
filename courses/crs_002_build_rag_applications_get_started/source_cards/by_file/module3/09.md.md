## RAG recap (whole course)

1. Chunk + embed prompts and documents (BERT/GPT-family embedders)
2. **Retrieve** by distance (dot product, cosine similarity)
3. **Augmented query** = user prompt + retrieved content
4. **Generate** informed answer

**Two components:** Retriever + Generator.

Why RAG: accurate, domain-specific, up-to-date **without retraining**.

## LlamaIndex section (verbatim ideas from video)

### Framework

- Context augmentation: ingestion, chunking, indexing, retrieval
- Use cases: RAG Q&A, chatbots, document understanding, extraction

### vs LangChain

- LlamaIndex: built-in loaders + **query engines**
- LangChain: chaining workflows

### Conversational RAG design

```text
ingest → chunk to nodes → embed/store vectors → query engine / synthesizer → responses
```

### Loading (`01.md` content repeated)

- Formats: text, PDF, MD, CSV, JSON, HTML
- `Document`: id, embedding slot, metadata, relationships, text
- `SimpleDirectoryReader`: folder, recursive, specific files, `required_exts`
- `SentenceSplitter` + semantic / LangChain wrappers
