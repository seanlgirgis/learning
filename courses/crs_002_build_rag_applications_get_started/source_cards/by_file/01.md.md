## Section 1 ‚Äî What Gradio is

- Open-source **Python** library
- Builds **web-based UIs** for ML models and computational tools
- Emphasis: **ease of use** ‚Äî demos without writing HTML/CSS/JS

---

## Section 2 ‚Äî Four setup steps (memorize)

| Step | What you do |
|------|-------------|
| 1 | Write Python ‚Äî functions and logic |
| 2 | Create interface ‚Äî `gr.Interface` specifies **inputs** and **outputs** for those functions |
| 3 | Configure how users interact (labels, components) |
| 4 | `launch()` ‚Äî starts local server; open **local or public URL** |

Install: `pip install gradio` ‚Üí `import gradio as gr`

---

## Section 3 ‚Äî First demo: text in ‚Üí text out

**Core class:** `gr.Interface` ‚Äî wraps a Python function with a web UI.

| Piece | Role |
|-------|------|
| Your `fn` | Runs when user clicks Submit |
| `gr.Textbox` | Text input **and** text output (label each one) |
| `demo.launch()` | Starts server |

## Section 4 ‚Äî Multiple inputs

- `gr.Number` ‚Äî numeric field (like Textbox but for numbers)
- Pass **list** of components: `[gr.Textbox(), gr.Number()]`
- **Order of list = order of function parameters**

Example mental model:

```python
def fn(text_value, number_value):
    ...

inputs=[gr.Textbox(), gr.Number()]
```

---

## Section 5 ‚Äî File upload

- `gr.File` ‚Äî upload or drop files in the browser
- Supports **multiple** uploads
- Backend function receives **paths** to temp files ‚Äî you process them in Python
- Typical pattern: count files, read paths, load into your pipeline
