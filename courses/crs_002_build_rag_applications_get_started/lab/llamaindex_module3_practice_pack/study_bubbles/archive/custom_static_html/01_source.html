Pick a source type on the map.

Practice pack: sources live in ../data/company_knowledge/ until a loader runs.

Source (hub)

Any place your app’s knowledge lives before LlamaIndex touches it.

Boundary: Source is outside the framework. First LlamaIndex object = Document from a Loader.

Files

Single files on disk: .md, .txt, .pdf, .csv, .json.

../data/company_knowledge/01_company_overview.md ../data/company_knowledge/08_product_matrix.csv

Folders

A directory of many files — typical RAG corpus layout.

../data/company_knowledge/ 01_company_overview.md 02_support_policy.md ...

In-memory text

String already in Python — no file path needed.

text = "Northwind refund policy: 30 days."
Often wrapped manually in Document(text=...) — skips file loader.

Web page

Content fetched from a URL at runtime.

# pseudo-code SimpleWebPageReader().load_data([url])

API

HTTP JSON — e.g. mock LinkedIn profile in icebreaker lab.

data = response.json() # source is the JSON blob # later: Document(text=json.dumps(data))

Database

Tables or query results — each row or joined text can become a Document.

# pseudo-code for row in rows: Document(text=row.body, metadata={"id": row.id})

Cloud storage

Blobs in S3 / GCS / Azure — usually download or use a connector, then load.

Pattern: cloud blob → local path or stream → loader.

Source ≠ Document

  • Source: file on disk, API JSON, DB row — outside LlamaIndex
  • Document: Python object with .text + .metadata
Loader is the bridge between these two worlds.
