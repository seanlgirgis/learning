# Data Handling Policy

## Data Classes

Public data may be shared externally.

Internal data may be shared with employees and approved contractors.

Confidential data includes employee records, customer contracts, private architecture notes, and unpublished financial details.

Restricted data includes secrets, passwords, API keys, private tokens, regulated customer data, and production credentials.

## AI Tool Rules

Employees may use approved AI tools for summarization, drafting, and code explanation.

Employees must not paste restricted data into any AI tool.

Before uploading confidential documents into an AI tool, employees must confirm that the tool is approved for confidential data and that the document does not contain restricted data.

If there is doubt, remove sensitive sections or ask the security team.

## Retention

Temporary working files should be deleted after the task is complete.

Approved knowledge base files should include a source, owner, and last-reviewed date.
