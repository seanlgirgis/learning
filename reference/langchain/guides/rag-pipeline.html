Guide · Labs 23–31

RAG pipeline end-to-end

One spine: load → split → embed → store → retrieve → answer. No LLM until the last step.

Key take — six steps

StepActionModuleLab
1loader.load()1223–25
2splitter.split_documents()1326–27
3make_watsonx_embeddings()1428
4Chroma.from_documents()1529
5as_retriever() or similarity_search1630
6RetrievalQA.invoke(question)1631

Memory: ingest first, LLM last. Embeddings are Route B (watson_llm), not generate().

Flow diagram

PDF / Web URL
    → PyPDFLoader / WebBaseLoader
    → list[Document]
    → RecursiveCharacterTextSplitter
    → chunks
    → make_watsonx_embeddings + Chroma.from_documents
    → vector_store
    → retriever.invoke(question) OR RetrievalQA
    → answer str (grounded in chunks)

Minimal working chain (Lab 31)

from langchain_classic.chains import RetrievalQA
from langchain_community.document_loaders import PyPDFLoader
from langchain_community.vectorstores import Chroma
from langchain_text_splitters import RecursiveCharacterTextSplitter
from watson_llm import make_watsonx_embeddings, make_watsonx_llm

loader = PyPDFLoader(PDF_URL)
chunks = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50).split_documents(loader.load())
store = Chroma.from_documents(chunks, make_watsonx_embeddings())
qa = RetrievalQA.from_chain_type(
    llm=make_watsonx_llm(),
    retriever=store.as_retriever(),
    chain_type="stuff",
)
qa.invoke("what is this paper discussing?")

Parent document variant (Lab 32)

Search small child chunks; return large parent for LLM context. See Module 16 · 32.parent_document_retriever.py.

Traps

  • Skipping split — whole PDF as one vector → useless search.
  • Web URL for factual PDF QA — Lab 31 uses LangChain paper PDF on purpose; random URLs → weak answers.
  • Confusing RAG with memory — RAG = external docs; memory = chat turns (17).
  • langchain_classic — RetrievalQA import in 1.x venv.
