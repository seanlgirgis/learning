ReAct agent internals

Chain = you wire fixed steps. Agent = LLM writes Thought → picks Action → reads Observation → repeats.

Key take — chain vs agent

Chain / LCELAgent (ReAct)
StepsFixed at build timeLLM chooses each turn

Building blocks (Lab 35)

  1. LLM — make_watsonx_llm()
  2. Tools — Tool(...) or @tool with clear descriptions
  3. Prompt — ReAct template with {tools}, {tool_names}, {input}
  4. Agent — create_react_agent(llm, tools, prompt)
  5. Executor — AgentExecutor(agent=..., tools=..., verbose=True)

Minimal skeleton

from langchain_classic.agents import AgentExecutor, create_react_agent
from langchain_core.prompts import PromptTemplate
from watson_llm import make_watsonx_llm

react_prompt = PromptTemplate.from_template("""... ReAct format ...
Tools: {tools}
Tool names: {tool_names}
Question: {input}
{agent_scratchpad}""")

agent = create_react_agent(make_watsonx_llm(), tools, react_prompt)
executor = AgentExecutor(agent=agent, tools=tools, verbose=True)
executor.invoke({"input": "your question"})

Full script: playground/langchain/35.agents.py

Traps

  • Agent for fixed workflows — use LCEL; agents are slower and less predictable.
  • Vague tool descriptions — the description is how the LLM picks tools.
  • PythonREPL — runs real code (langchain_experimental).
  • Confusing with RAG — RAG chain retrieves docs; agent may call a RAG tool you define (RAG guide).
