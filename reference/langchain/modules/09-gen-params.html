Generation parameters

Knobs for how the model writes — temperature, length, sampling. Passed when you build the LLM, not between invoke calls.

Key take (one rule)

Params are frozen at make_watsonx_llm({...}) time. Want different behavior? Build a new LLM.

PatternCode
One setting llm = make_watsonx_llm({GenParams.MAX_NEW_TOKENS: 80})
Compare two behaviors llm_creative = make_watsonx_llm({GenParams.TEMPERATURE: 0.8})
llm_precise = make_watsonx_llm({GenParams.TEMPERATURE: 0.1})

Memory hook: new behavior → new LLM. Import: from watson_llm import GenParams, make_watsonx_llm (re-exports IBM GenTextParamsMetaNames).

Job (help-desk story)

Thermostat on the HVAC unit — set when you install it, not adjusted mid-conversation. Creative poem mode (warm) vs fact mode (cool) = two units with different dials, same building badge (WATSONX_* env).

Problem without explicit params

  • Every lab silently uses watson_llm.py defaults — you cannot explain why answers ramble or repeat.
  • Tweaking “the same LLM” between invokes does nothing — params are baked in at construction.
  • Coursera compares Granite vs Llama; your local exercise compares temperature on one model (WATSONX_MODEL_ID).

Lifecycle

  1. Import — GenParams + make_watsonx_llm from watson_llm.
  2. Choose overrides — dict keys like GenParams.TEMPERATURE merge over defaults in watson_llm.py.
  3. Build — llm = make_watsonx_llm({...}) → WatsonxLLM with frozen params.
  4. Use — llm.invoke(...) or prompt | llm | parser — same params every call on that object.
  5. Compare — second LLM with a different dict; same prompts, side-by-side (Lab 18).

Flow

from watson_llm import GenParams, make_watsonx_llm

DEFAULT_PARAMS in watson_llm.py  (256 tokens, temp 0.5, TOP_K 1, …)
        │
        ├─ make_watsonx_llm({TEMPERATURE: 0.8})  →  llm_creative
        └─ make_watsonx_llm({TEMPERATURE: 0.1})  →  llm_precise
                    │
                    ▼
        llm_creative.invoke("Write a poem…")
        llm_precise.invoke("Write a poem…")     ← same prompt, different frozen dials

Knobs we use

ParamEffectIf wrong then…
TEMPERATURE Low (0.1) tighter · high (0.8) more varied With default TOP_K: 1, 0.8 vs 0.1 may look identical
MAX_NEW_TOKENS Caps answer length Model cuts off mid-sentence or rambles past what you wanted
TOP_K / TOP_P How wide the sampling pool is TOP_K: 1 = greedy — temperature barely bites; loosen for Lab 18 demos
MIN_NEW_TOKENS Floor on generation length Rarely touched in your labs; default 0

Defaults in playground/langchain/watson_llm.py: MAX_NEW_TOKENS: 256, TEMPERATURE: 0.5, TOP_P: 0.2, TOP_K: 1.

Minimal example

from watson_llm import GenParams, make_watsonx_llm

llm = make_watsonx_llm({GenParams.MAX_NEW_TOKENS: 80})
print(llm.invoke("One sentence about cats."))

Full lab (Lab 18 — temperature compare)

import os
from watson_llm import GenParams, make_watsonx_llm

print("Model:", os.environ["WATSONX_MODEL_ID"])

llm_creative = make_watsonx_llm({
    GenParams.TEMPERATURE: 0.8,
    GenParams.MAX_NEW_TOKENS: 80,
})
llm_precise = make_watsonx_llm({
    GenParams.TEMPERATURE: 0.1,
    GenParams.MAX_NEW_TOKENS: 80,
})

prompts = [
    "Write a short poem about artificial intelligence",
    "What are the key components of a neural network?",
    "List 5 tips for effective time management",
]

for prompt in prompts:
    print("PROMPT:", prompt)
    print("creative:", llm_creative.invoke(prompt))
    print("precise:", llm_precise.invoke(prompt))

Traps (from Lab 18)

  • Params not per invoke — llm.invoke(prompt, temperature=0.9) is not the pattern. Build make_watsonx_llm({GenParams.TEMPERATURE: 0.9}) instead.
  • TOP_K: 1 mutes temperature — creative vs precise outputs may match. Loosen TOP_K / TOP_P in overrides to see temperature bite.
  • Wrong compare — local exercise is same model, two temps — not Granite vs Llama (Coursera cloud setup).
  • make_llm typo — use make_watsonx_llm from watson_llm (07).

Recall drills

How do you get creative (0.8) vs precise (0.1) on the same model?
Two make_watsonx_llm({GenParams.TEMPERATURE: …}) calls — two LLM objects. Same invoke prompts on each.
When are GenParams applied — at invoke or at build?
At build (make_watsonx_llm({...})). Frozen for every invoke on that object.
0.8 vs 0.1 looks identical — first thing to check?
Default TOP_K: 1 in watson_llm.py — greedy sampling; try higher TOP_K or TOP_P in your override dict.
