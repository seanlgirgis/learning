Agents + tools

Chain = fixed steps you wire. Agent = LLM picks tools in a loop — Thought → Action → Observation (ReAct).

Key take

PieceRole
Tool / @toolNamed function the agent can call (calculator, weather, format text)
ReAct promptTeaches Thought / Action / Action Input / Observation format
create_react_agentLLM + tools + prompt → agent Runnable
AgentExecutorRuns the loop until final answer or max iterations

Memory hook: chain = fixed · agent = LLM chooses tools.

Job (help-desk story)

Generalist with a tool belt. You do not script “step 1 calculator, step 2 weather.” The agent reads the question, picks a tool, reads the result, repeats until done.

Flow (ReAct loop)

User question
        │
        ▼
   Thought: "I need math"
   Action: Python_REPL
   Action Input: "1234 * 5678"
        │
        ▼
   Observation: 7006652
        │
        ▼
   Thought: "I have the answer"
   Final Answer: ...

Traps

  • Agent when a chain suffices — slower, less predictable; use pipes for fixed workflows.
  • Bad tool descriptions — agent picks wrong tool; description is the API doc for the LLM.
  • langchain_experimental — PythonREPL runs real code — sandbox awareness.
  • langchain_classic.agents — import path in your 1.x venv.

Recall drills

Chain vs agent in one line?
Chain: you wire steps. Agent: LLM chooses tools in a loop until done.
What is ReAct?
Reason + Act — alternate Thought, Action (tool call), Observation until Final Answer.
Two building blocks after tools + prompt?
create_react_agent + AgentExecutor.
