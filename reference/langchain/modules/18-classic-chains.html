Classic chains

Wire multi-step pipelines where step 2 uses step 1's output. Coursera 0.x style (LLMChain, SequentialChain) alongside modern LCEL (03).

Key take — two styles, same pipeline

StyleAPIWhen
Classic LLMChain + SequentialChain(chains=[...], input_variables=..., output_variables=...) Notebook / Exercise 6 twin
LCEL RunnablePassthrough.assign(...) + prompt | llm | parser Your default for new code

Memory hook: classic = named chains · LCEL = pipes + assign. Lab 34 builds both for same meal/recipe and review pipelines.

Job (help-desk story)

Assembly line with named stations. Station 1 (location → dish) passes its output as input to station 2 (dish → recipe). SequentialChain is the conveyor belt wiring; LCEL is the same line with pipe syntax.

Pipelines in Lab 34

  • Notebook demo — location → classic dish → recipe → cook time
  • Exercise 6 — review → sentiment → summary → customer response

Flow (SequentialChain)

input: {"location": "China"}
        │
        ▼
   LLMChain (meal_prompt)  →  meal
        │
        ▼
   LLMChain (recipe_prompt)  →  recipe  (uses meal)
        │
        ▼
   output_variables: meal, recipe, time, ...

from langchain_classic.chains import LLMChain, SequentialChain

meal_chain_classic = LLMChain(llm=llm, prompt=meal_prompt)

overall_chain = SequentialChain(
    chains=[chain_one, chain_two, chain_three],
    input_variables=["location"],
    output_variables=["meal", "recipe", "time"],
    verbose=True,
)

Traps

  • langchain_classic — LLMChain, SequentialChain — not 0.x langchain.chains paste.
  • Output key names — next chain prompt must reference correct output_key from previous step.
  • Chain ≠ agent — steps are frozen; agent loops with tool choice (19).

Recall drills

What does SequentialChain add over one LLMChain?
Runs multiple chains in order — later prompts use earlier outputs as inputs.
Lab 34 builds two styles for what reason?
Same pipeline in classic API (notebook) and LCEL (your forward path).
Chain vs agent?
Chain: fixed steps. Agent: LLM picks tools in a ReAct loop.
