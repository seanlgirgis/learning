Memory

LLMs are stateless — memory stores past turns and attaches them to the next call. Upgrades 08 (you pasted fake history) with real persistence patterns.

Key take — two levels

LevelWhatWho appends turns
A — manual ChatMessageHistory + chat.invoke(history.messages) You add each HumanMessage / AIMessage
B — automatic ConversationBufferMemory + ConversationChain Chain updates memory each predict

Memory hook: manual list vs buffer memory on chain. Not RAG — conversation turns, not document chunks.

Job (help-desk story)

Call-center CRM. Without memory, every call starts cold. Memory is the ticket history the next agent (LLM) reads before replying — “Who am I?” only works if prior turns were saved.

Problem without it

  • Lab 17 — you hand-build AIMessage fakes; tedious for real chat.
  • Each invoke sees only what you send — model forgets “I’m a little cat” on the next line unless memory wires it in.
  • Confusing chat history with RAG — different data (turns vs documents).

Flow (Lab 33)

User: "My name is Alice"
        │
        ▼
ConversationChain.predict(input="...")
        │
        ▼
ConversationBufferMemory saves Human + AI lines
        │
        ▼
Next predict → memory injected into prompt → "Who am I?" works

Full lab (33)

from langchain_community.chat_message_histories import ChatMessageHistory
from langchain_classic.memory import ConversationBufferMemory
from langchain_classic.chains import ConversationChain
from watson_llm import make_watsonx_llm

# Manual
history = ChatMessageHistory()
history.add_user_message("Hi!")
history.add_ai_message("Hello!")
chat.invoke(history.messages)

# Automatic
memory = ConversationBufferMemory()
chain = ConversationChain(llm=make_watsonx_llm(), memory=memory, verbose=True)
chain.predict(input="I have a little cat")
chain.predict(input="Who am I?")  # Exercise 5 / little-cat demos

Recall drills

Why do LLMs need memory?
Stateless — each call only sees current input unless you attach prior turns.
Manual vs automatic in Lab 33?
Manual: ChatMessageHistory. Automatic: ConversationBufferMemory on ConversationChain.
Memory vs RAG?
Memory = conversation history. RAG = retrieve document chunks for facts.
