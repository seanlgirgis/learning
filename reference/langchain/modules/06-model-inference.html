ModelInference

IBM’s low-level watsonx client — not LangChain. String prompt in, dict JSON out; you slice generated_text yourself.

Job (help-desk story)

Direct line to IBM support — no LangChain receptionist. You dial model.generate(), IBM sends a full JSON ticket back; you read the answer from results[0].generated_text.

Problem without it / when to use it

  • Use: Coursera LOCAL SETUP cells, Lab 15 “is watsonx wired?”, when you want the raw IBM response shape.
  • Use: Scripts with no LangChain — one prompt, one completion, print text.
  • Do not use: Inside prompt | ??? | parser — use WatsonxLLM instead.
  • Do not use: When you already have LCEL chains — extra dict-slicing code for no benefit.

Object lifecycle

  1. Build — ModelInference(model_id=..., credentials=..., project_id=..., params=...) or import model from watson_helper after set_env.ps1.
  2. Configure — params uses GenTextParamsMetaNames (same family as Module 02). Credentials dict: url, api_key.
  3. Invoke — msg = model.generate("plain string prompt") — not invoke, not a template dict.
  4. Output — dict → extract msg["results"][0]["generated_text"] → str for your app.

Data flow

"In today's sales meeting, we "
        │
        ▼
┌─────────────────────┐
│   ModelInference    │  IBM text generation API
│   .generate(str)    │
└──────────┬──────────┘
           ▼
    dict (full JSON)
    └─ results[0]
         └─ generated_text  → "discussed Q3 targets..."
           │
           ▼
      print(text)   ← you slice this yourself

Minimal example

from watson_helper import model

msg = model.generate("In today's sales meeting, we ")
text = msg["results"][0]["generated_text"]
print(text)

Traps (from your labs)

  • prompt | model | parser — ModelInference is not a Runnable. Same env vars, wrong object for LCEL. Use make_watsonx_llm() or watson_llm.make_watsonx_llm().
  • Forgetting the slice — print(msg) dumps a huge dict. You almost always want msg["results"][0]["generated_text"].
  • Import before set_env.ps1 — watson_helper.model can be None or creds empty; run env script in the same shell first.
  • Assuming OpenAI JSON — Watson uses results / generated_text; copy the Lab 15 path, not ChatGPT docs.
