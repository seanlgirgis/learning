Module 07 · Infra

Watson helper files

Small modules that read WATSONX_* once — you import and plug in, no Skills Network credentials in every cell.

Two plugs, same env

set_env.ps1  →  WATSONX_MODEL_ID, URL, PROJECT_ID, APIKEY
        ├─ watson_helper / coursera_watsonx_model  →  model (ModelInference)
        └─ watson_llm / coursera_llm_model         →  make_*_llm() (WatsonxLLM)

File map

FileFolderBuildsUse when
watson_helper.py playground/langchain/ model (ModelInference) Lab scripts · model.generate()
langchain_helper.py playground/langchain/ make_llm() Lab scripts · notebook twin (llama_llm)
watson_llm.py playground/langchain/ make_watsonx_llm() Lab scripts · full LangChain API
coursera_watsonx_model.py playground/notebooks/ model + credentials Jupyter · LOCAL SETUP cell
coursera_llm_model.py playground/notebooks/ make_llm() Jupyter · llama_llm = make_llm()
coursera_embeddings.py playground/notebooks/ make_embeddings() Jupyter · RAG / vector store cells

What each helper exports

IBM path — watson_helper

from watson_helper import model, model_id, credentials, parameters, project_id

model is ready. Call model.generate(...). See Module 06.

LangChain path — langchain_helper (notebook twin)

from langchain_helper import make_llm, GenParams

llama_llm = make_llm()
llama_llm = make_llm({GenParams.TEMPERATURE: 0.8})

Same API as coursera_llm_model in Jupyter. Under the hood calls watson_llm.make_watsonx_llm().

llm is WatsonxLLM. Use in prompt | llm | parser. See Module 02.

Notebook name trap

The LangChain lab uses llama_llm — Coursera’s variable name for a Llama model. Locally it is still your WATSONX_MODEL_ID (e.g. Mistral). The name is kept so later cells run unchanged.
