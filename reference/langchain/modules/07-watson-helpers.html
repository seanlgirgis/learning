Watson helper files

One env load (set_env.ps1) → pick the right plug-in file. Stops you pasting IBM credentials into every lab and notebook cell.

Two routes (key take)

Everything in this module hangs on one choice:

RouteImportHow you call itWhat it is
A from watson_helper import model model.generate(...) Lower-level IBM ModelInference — raw dict back
B from watson_llm import make_watsonx_llm .invoke(...) or prompt | llm | parser LangChain Runnable — pipes and chains

Memory hook: helper → generate · llm → invoke / pipe. Notebooks and .py labs use the same two files (start_jupyter.ps1 adds playground/langchain to PYTHONPATH). Spatial map: Watson Two Routes bubble.

Job (help-desk story)

Building security desk. set_env.ps1 issues the badge (WATSONX_*). Two doors only: IBM SDK vs LangChain — same badge, different client.

Problem without helpers

  • Every script repeats credentials — drift and copy-paste errors.
  • Easy to mix ModelInference and WatsonxLLM in one pipe.
  • Multiple alias files trained the wrong mental model — project now uses two files only.

Lifecycle (pick a route)

  1. Env — set_env.ps1 (Jupyter: start_jupyter.ps1 does this + PYTHONPATH).
  2. Choose route — need generate() / IBM dict? → Route A. Need invoke or pipes? → Route B.
  3. Import — model (A) or make_watsonx_llm() / make_watsonx_embeddings() (B).
  4. Use — A: slice generated_text from dict. B: invoke or prompt | llm | parser.

Flow (same env, two doors)

set_env.ps1  →  WATSONX_*
        │
        ├─ Route A: watson_helper.py
        │     from watson_helper import model
        │     model.generate(str) → dict
        │
        └─ Route B: watson_llm.py
              from watson_llm import make_watsonx_llm
              llm = make_watsonx_llm()
              llm.invoke(...)  or  prompt | llm | parser
              make_watsonx_embeddings()  →  RAG (still Route B)

Notebooks + .py labs: same files (PYTHONPATH = playground/langchain)

File map (two files)

FileRouteExportsUse when

Minimal examples

# Route A — IBM generate (Lab 15)
from watson_helper import model
print(model.generate("Hello")["results"][0]["generated_text"])

# Route B — LangChain invoke (Lab 16)
from watson_llm import make_watsonx_llm
llama_llm = make_watsonx_llm()
print(llama_llm.invoke("Who is man's best friend?"))

# Route B — LCEL pipe (Lab 08+)
from watson_llm import make_watsonx_llm
llm = make_watsonx_llm()
chain = prompt | llm | StrOutputParser()

Confused with

NameWhat it isNot
watson_helper Route A — IBM ModelInference Not watson_llm — similar names, different route
watson_llm Route B — LangChain builders Not low-level generate()
llama_llm Variable name in notebooks Not your model family — use WATSONX_MODEL_ID

Imports (notebooks + labs — same)

# Route A
from watson_helper import model, model_id, credentials

# Route B
from watson_llm import GenParams, make_watsonx_llm, llm_model
from watson_llm import make_watsonx_embeddings  # RAG labs

llama_llm = make_watsonx_llm()  # variable name optional

Traps

  • prompt | model | parser — model from Route A is not Runnable. Use Route B: make_watsonx_llm().
  • watson_helper vs watson_llm — sound alike; Route A vs Route B.
  • Import before env — run set_env.ps1 first.
  • Jupyter without start_jupyter.ps1 — PYTHONPATH must include playground/langchain.

Recall drills

Lab 15 vs Lab 16 — which route?
Lab 15 → Route A · watson_helper · generate. Lab 16 → Route B · watson_llm · invoke.
Where do embeddings come from?
from watson_llm import make_watsonx_embeddings — still Route B (LangChain).
Four env vars every helper expects?
WATSONX_MODEL_ID, WATSONX_URL, WATSONX_PROJECT_ID, WATSONX_APIKEY — from set_env.ps1.
