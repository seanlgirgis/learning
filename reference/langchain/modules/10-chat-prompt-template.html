ChatPromptTemplate

01 mail-merge for one string. This module mail-merges role messages — system + user — so you can pipe prompt | llm without hand-building lists every time.

Two patterns (key take)

Same object (ChatPromptTemplate.from_messages([...])). Pick the list shape:

PatternList containsInvoke dictLab 20 section
A — role tuples ("system", "..."), ("user", "... {topic}") {"topic": "cats"} 1 / 1b
B — MessagesPlaceholder ("system", "..."), MessagesPlaceholder("msgs") {"msgs": [HumanMessage(...)]} 2 / 2b

Memory hook: {holes} in tuples · placeholder = slot for a message list. Upgrades 08 (hand-built lists) for reusable pipes.

Job (help-desk story)

Pre-printed form with fixed headers: From: System and From: User. You only fill the blanks ({topic}) or drop in a whole attachment (msgs list). The template assembles the thread; the LLM desk reads the finished packet.

Problem without it

  • Module 08 works but you rebuild SystemMessage / HumanMessage in code every call — no reusable pipe.
  • PromptTemplate outputs one string — weak for system vs user roles in chat models.
  • Invoke dict keys must match every {hole} and every MessagesPlaceholder name — errors before the LLM runs.

Lifecycle

  1. Build template — ChatPromptTemplate.from_messages([...]) at setup.
  2. Prepare dict — keys for {variables} and/or placeholder names (msgs).
  3. Format only — prompt.invoke(dict) → ChatPromptValue (message list preview).
  4. Pipe to model — chain = prompt | llm then chain.invoke(dict) → str.

Minimal example (Pattern A)

from langchain_core.prompts import ChatPromptTemplate
from watson_llm import make_watsonx_llm

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant."),
    ("user", "Tell me a joke about {topic}"),
])
chain = prompt | make_watsonx_llm()
print(chain.invoke({"topic": "cats"}))

Full lab (Lab 20 — both patterns)

from langchain_core.messages import HumanMessage
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from watson_llm import GenParams, make_watsonx_llm

llama_llm = make_watsonx_llm({GenParams.MAX_NEW_TOKENS: 120})

# Pattern A — format, then pipe
joke_prompt = ChatPromptTemplate.from_messages([
    ("system", "Helpful assistant. One response only."),
    ("user", "Tell me a joke about {topic}"),
])
joke_input = {"topic": "cats"}
print(joke_prompt.invoke(joke_input))           # ChatPromptValue
print((joke_prompt | llama_llm).invoke(joke_input))  # str

# Pattern B — placeholder for message list
history_prompt = ChatPromptTemplate.from_messages([
    ("system", "Answer only the user request."),
    MessagesPlaceholder("msgs"),
])
history_input = {"msgs": [HumanMessage(content="What is the day after Tuesday?")]}
print((history_prompt | llama_llm).invoke(history_input))

Traps (from Lab 20)

  • Format vs send — prompt.invoke(dict) shows ChatPromptValue; it does not call the model. Use (prompt | llm).invoke(dict) for an answer.
  • Placeholder key must match — MessagesPlaceholder("msgs") requires {"msgs": [...]}, not {"messages": [...]}.
  • List not string — placeholder value must be a list of message objects (e.g. HumanMessage), not plain text.
  • Wrong prompt type in pipe — mixing PromptTemplate and chat tuples in one template without planning — pick string template (01) or chat template (10) for the chain.

Recall drills

Pattern A vs B — what goes in from_messages([...])?
A: ("system", …) + ("user", "… {topic}"). B: ("system", …) + MessagesPlaceholder("msgs").
prompt.invoke vs (prompt | llm).invoke?
prompt.invoke → formatted messages only. (prompt | llm).invoke → messages sent to model → str reply.
08 vs 10 — when use ChatPromptTemplate?
08: one-off hand-built message lists. 10: reusable template + dict, especially in prompt | llm pipes (Lab 20).
