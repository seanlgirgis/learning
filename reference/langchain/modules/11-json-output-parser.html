JsonOutputParser

Last step in the pipe when you need a dict, not plain text. Tell the model the JSON shape in the prompt; parser turns the reply into Python fields.

Two paths (key take)

Same pipe ending: prompt | llm | json_parser → dict. How you define the shape:

PathParserFormat instructionsLab
A — Pydantic JsonOutputParser(pydantic_object=Joke) parser.get_format_instructions() — auto from class 21 Part A
B — hand-written JsonOutputParser() You write the JSON rules string yourself 22 (Exercise 2)

Memory hook: class → auto instructions · bare parser → you write rules. vs 04: StrOutputParser → str; this → dict.

Job (help-desk story)

Export desk at the end of the line. The specialist (LLM) writes a messy reply; this step enforces a form with named boxes (setup, punchline, title, …) and hands your app a dict it can use in code, a DB, or an API.

Problem without it

  • Model wraps JSON in markdown fences or adds chit-chat — json.loads on raw text breaks.
  • You copy-paste format rules into every prompt; drift when fields change.
  • StrOutputParser only gives str — you still regex or parse by hand for structured data.
  • Downstream code wants result["title"] — need a parser step that normalizes LLM output to dict.

Lifecycle (four steps)

  1. Define shape — Pydantic BaseModel (Path A) or hand-written JSON spec (Path B).
  2. Build parser — JsonOutputParser(pydantic_object=...) or bare JsonOutputParser().
  3. Inject instructions — get_format_instructions() or your string → partial_variables on PromptTemplate.
  4. Pipe — prompt | make_watsonx_llm() | parser → chain.invoke({"query": ...}) returns dict.

from pydantic import BaseModel, Field
from langchain_core.output_parsers import JsonOutputParser

Minimal example (Path A)

class Joke(BaseModel):
    setup: str = Field(description="question to set up a joke")
    punchline: str = Field(description="answer to resolve the joke")

parser = JsonOutputParser(pydantic_object=Joke)
prompt = PromptTemplate(
    template="Answer the user query.\n{format_instructions}\n{query}\n",
    input_variables=["query"],
    partial_variables={"format_instructions": parser.get_format_instructions()},
)
chain = prompt | make_watsonx_llm() | parser
chain.invoke({"query": "Tell me a joke."})  # dict

Lab 22 — Movie (hand-written)

json_parser = JsonOutputParser()
format_instructions = """RESPONSE FORMAT: Return ONLY a single JSON object…
{"title": "...", "director": "...", "year": 2000, "genre": "..."}"""

prompt_template = PromptTemplate(
    template="Task: movie \"{movie_name}\" in JSON.\n{format_instructions}",
    input_variables=["movie_name"],
    partial_variables={"format_instructions": format_instructions},
)
movie_chain = prompt_template | make_watsonx_llm() | json_parser
result = movie_chain.invoke({"movie_name": "The Matrix"})
print(result["title"])

Traps (from Labs 21–22)

  • Wrong Pydantic import — your venv: from pydantic import BaseModel, Field. Course may show langchain_core.pydantic_v1.
  • Instructions not in prompt — without {format_instructions} in template + partial_variables, model free-forms JSON → parse fails.
  • Markdown fences — model returns ```json ... ``` — tighten prompt (“ONLY JSON, no markdown”) like Lab 22.
  • Parser before LLM — last step only: prompt | llm | parser.

Recall drills

Path A vs B — how do format instructions get into the prompt?
A: JsonOutputParser(pydantic_object=Joke).get_format_instructions(). B: you write the format_instructions string yourself for bare JsonOutputParser().
What does chain.invoke return — str or dict?
dict (e.g. {"setup": "...", "punchline": "..."}). StrOutputParser would return str.
Where does format_instructions live in the template setup?
partial_variables={"format_instructions": ...} on PromptTemplate — baked at build, not passed each invoke.
