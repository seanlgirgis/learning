Text splitters

Scissors on Document lists — big pages → smaller chunks with overlap so retrieval does not cut sentences in half blindly.

Key take — two splitters

SplitterHow it cutsTypical useLab
CharacterTextSplitter Fixed size on a separator (e.g. \n) Teaching / simple PDF chunks 26
RecursiveCharacterTextSplitter Tries paragraph → line → word hierarchy RAG pipelines (29–31) 27, 29+

Memory hook: chunk_size · chunk_overlap · split_documents(list) → more Documents.

Job (help-desk story)

Paper shredder with adjustable strip width and overlap. Whole reports (loaders) go in; uniform strips (chunks) come out — sized for embedding and retrieval.

Problem without it

  • Embedding a 50-page PDF as one vector — search is useless.
  • Chunks too large — exceed model context when stuffed into RAG prompt.
  • Zero overlap — answers lose context split across chunk boundaries.

Lifecycle

  1. Load — documents = loader.load() from 12.
  2. Configure splitter — chunk_size, chunk_overlap, optional separator.
  3. Split — chunks = splitter.split_documents(documents).
  4. Inspect — Lab 27 compares chunk counts for two splitter configs on same PDF.

Flow

list[Document]  (6 PDF pages)
        │
        ▼
CharacterTextSplitter(chunk_size=200, chunk_overlap=20)
        │
        ▼
list[Document]  (47 chunks)  — same metadata carried over

Knobs + if wrong then…

ParamEffectIf wrong then…
chunk_sizeTarget max chars per chunkToo big → poor retrieval; too small → fragmented meaning
chunk_overlapShared tail/head between neighbors0 overlap → lost context at edges
separatorPreferred break (Character splitter)Wrong separator → mid-word cuts
from langchain_text_splitters import CharacterTextSplitter, RecursiveCharacterTextSplitter

Minimal example (Lab 26)

splitter = CharacterTextSplitter(chunk_size=200, chunk_overlap=20, separator="\n")
chunks = splitter.split_documents(documents)
print(len(chunks), chunks[10].page_content)

Full lab (Lab 27 — compare splitters)

splitter_1 = CharacterTextSplitter(chunk_size=300, chunk_overlap=30)
splitter_2 = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
chunks_1 = splitter_1.split_documents(pdf_document)
chunks_2 = splitter_2.split_documents(pdf_document)

Traps

  • Same PDF, different counts — Lab 27: splitter choice changes chunk count — not a bug.
  • Skipping load step — splitter needs list[Document] from loader first.

chunk_size and chunk_overlap.
