Module 02

WatsonxLLM

Configured client to IBM watsonx — the “phone” to the model, not the model on your PC.

When to use

Plain-text prompts (string in → string out). Step 2 in prompt | llm | parser. Run set_env.ps1 first.

Object

ObjectTypeRole
llmWatsonxLLMGenerate text via watsonx text API

Environment (connection)

VariablePurpose
WATSONX_MODEL_IDWhich model
WATSONX_URLRegion endpoint
WATSONX_PROJECT_IDYour project GUID
WATSONX_APIKEYAPI key

GenParams (generation)

ParamTypical rangeEffect
MAX_NEW_TOKENS10–512Max output length
MIN_NEW_TOKENS0–50Minimum output length
TEMPERATURE0–1Randomness (0 = tight)
TOP_P0–1Nucleus sampling — token pool size
TOP_K1–501 = almost greedy

Helper plug-in (preferred in labs)

from watson_llm import make_watsonx_llm, GenParams

llm = make_watsonx_llm()
llm = make_watsonx_llm({GenParams.TEMPERATURE: 0.8})

Notebook: from coursera_llm_model import make_llm then llama_llm = make_llm(). See Module 07 and LLM plug-in recipe.

Constructor pattern (inline)

from langchain_ibm import WatsonxLLM
from ibm_watsonx_ai.metanames import GenTextParamsMetaNames as GenParams

llm = WatsonxLLM(
    model_id=os.environ["WATSONX_MODEL_ID"],
    url=os.environ["WATSONX_URL"],
    project_id=os.environ["WATSONX_PROJECT_ID"],
    apikey=os.environ["WATSONX_APIKEY"],
    params={
        GenParams.MAX_NEW_TOKENS: 256,
        GenParams.TEMPERATURE: 0.5,
    },
)

Methods (we use)

llm.invoke("plain text prompt")

Standalone call — no pipe. Returns str.

In a pipe

Chain calls llm.invoke with formatted prompt from step 1.

Trap: Do not use watson_helper.model in prompt | llm | parser. Use make_watsonx_llm() here.
Note: ChatWatsonx is the chat API (messages) — later topic. WatsonxLLM = text completion API.
