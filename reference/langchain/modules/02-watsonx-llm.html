WatsonxLLM

Configured client to IBM watsonx — the “phone line” to the model. Step 2 in prompt | llm | parser.

Job (help-desk story)

The specialist on the phone. Step 1 (prompt) hands them a finished paragraph; they reply with generated text. They do not fill forms or parse JSON — that is steps 1 and 3.

Problem without it

  • You would call IBM SDK model.generate() manually and dig generated_text out of a dict every time.
  • No Runnable interface — cannot use | in LCEL.
  • Env vars and GenParams duplicated in every script instead of watson_llm.py.

Object lifecycle

  1. Build — make_watsonx_llm() or WatsonxLLM(...) after set_env.ps1. Reads WATSONX_* env vars.
  2. Configure — params dict: MAX_NEW_TOKENS, TEMPERATURE, etc. Overrides merge with defaults in helper.
  3. Invoke — Standalone: llm.invoke("plain string"). In pipe: receives formatted prompt from step 1 automatically.
  4. Output — Usually str (text generation API). Pass to StrOutputParser or structured parser in step 3.

Minimal + full example

from watson_llm import make_watsonx_llm, GenParams

llm = make_watsonx_llm({GenParams.TEMPERATURE: 0.2})
print(llm.invoke("What is the capital of France?"))

chain = prompt | make_watsonx_llm() | StrOutputParser()
chain.invoke({"content": passage})

Traps (from your labs)

  • watson_helper.model in a pipe — That is ModelInference. Pipes need make_watsonx_llm() or watson_llm.make_watsonx_llm().
  • Forgot set_env.ps1 — All four WATSONX_* vars must exist before import/build; error looks like missing key, not “run env script”.
  • IBM SDK warnings — Mistral license + legacy API notices are normal; your labs filter them with warnings.filterwarnings on ibm_watsonx_ai.

Recall drills

What call do you use for a one-off string test vs inside prompt | llm | parser?
Same object: standalone llm.invoke("text"); in a pipe the framework calls llm.invoke with step 1’s output — you only call chain.invoke(dict).
Which object belongs in LCEL: WatsonxLLM or ModelInference?
WatsonxLLM (LangChain Runnable). ModelInference uses generate() and returns a dict.
What does raising TEMPERATURE do in Lab 18?
More varied wording run-to-run; same prompt can produce different answers. Low temperature (0.1) tightens output.
