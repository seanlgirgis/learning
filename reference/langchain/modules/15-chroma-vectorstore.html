Chroma vector store

Embed chunks, store vectors, search by meaning — similarity_search(query, k=3) returns the closest Documents.

Key take

docsearch = Chroma.from_documents(chunks, embedding_model)
hits = docsearch.similarity_search("What is LangChain?", k=3)

Memory hook: from_documents (build + store) · similarity_search (query → top-k chunks). Full ingest: load → split → embed → Chroma.

Job (help-desk story)

Filing cabinet indexed by meaning. Clerk (embeddings) tags each chunk; you ask a question and get the three closest folders — not alphabetical lookup.

RAG ingest pipeline (Labs 29+)

  1. WebBaseLoader / PyPDFLoader → documents (12)
  2. RecursiveCharacterTextSplitter → chunks (13)
  3. make_watsonx_embeddings() (14)
  4. Chroma.from_documents(chunks, embedding_model) — this module
  5. similarity_search or as_retriever() (16)

Full lab (29)

from langchain_community.vectorstores import Chroma
from watson_llm import make_watsonx_embeddings

vector_store = Chroma.from_documents(chunks, embedding_model)
hits = vector_store.similarity_search(query, k=3)
print(hits[0].page_content[:200])

Traps

  • Empty chunks — load/split failed → Chroma builds but search returns garbage.
  • k too small — miss context; too large — stuff too much into later RAG prompt.
  • Needs network + env — Lab 29 loads web URL and calls Watsonx embeddings.
