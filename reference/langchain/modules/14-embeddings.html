Embeddings

Text → vectors (lists of numbers). Similar meaning → nearby vectors. Still Route B: from watson_llm import make_watsonx_embeddings.

Key take

StepCallOutput
Buildemb = make_watsonx_embeddings(embed_params)WatsonxEmbeddings
Embed manyemb.embed_documents(texts)list[list[float]]
Embed queryemb.embed_query("...")One vector (Chroma search)

Memory hook: embeddings = meaning as numbers · same env as LLM · not generate() / not chat invoke.

Job (help-desk story)

GIS coordinates for sentences. Chunk text gets lat/long in meaning-space so “vector search” can find neighbors — keyword match not required.

Problem without it

  • Keyword search misses paraphrases (“mental health” vs “psychological wellness”).
  • Cannot store millions of chunks in prompt — need similarity lookup first.
  • Using LLM invoke for similarity — wrong tool; embeddings are cheaper and built for this.

Lifecycle

  1. Env — set_env.ps1 (same WATSONX_* as LLM).
  2. Build — make_watsonx_embeddings({EmbedTextParamsMetaNames...}).
  3. Embed chunks — embed_documents on strings or let Chroma call it (15).
  4. Search — embed question with embed_query, compare to stored vectors.

Flow

["LangChain helps...", "Mental health...", "Vector search..."]
        │

make_watsonx_embeddings()
        │
        ▼

Minimal + full lab (28)

from ibm_watsonx_ai.metanames import EmbedTextParamsMetaNames
from watson_llm import make_watsonx_embeddings

embed_params = {
    EmbedTextParamsMetaNames.TRUNCATE_INPUT_TOKENS: 3,
    EmbedTextParamsMetaNames.RETURN_OPTIONS: {"input_text": True},
}
watsonx_embedding = make_watsonx_embeddings(embed_params)

vectors = watsonx_embedding.embed_documents(texts)
print(len(vectors), len(vectors[0]))

Traps

  • LLM vs embedding model — RAG needs both; Lab 28 is embedding-only on purpose.
  • embed_documents vs embed_query — batch chunks vs single question string.
  • Import path — always from watson_llm import make_watsonx_embeddings in your playground.

Recall drills

Where do embeddings come from in your project?
from watson_llm import make_watsonx_embeddings — Route B, same env as LLM.
What does Lab 28 print for dimensions?
len(vectors[0]) — length of one embedding vector (model-dependent).
Why embed before Chroma?
Chroma needs vectors to run similarity search — text alone is not enough.
