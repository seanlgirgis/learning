JsonOutputParser + Joke

Pydantic class defines the JSON shape → parser auto-generates format instructions → pipe returns a dict with named fields.

1 — Problem

Your app needs structured data from the LLM — not a blob of text. A joke bot must return setup and punchline as separate fields for UI rendering or API export. Hand-parsing JSON from model output breaks when the model adds markdown fences or extra prose.

Not this recipe: comma-separated lists (Lab 21 Part B), hand-written JSON rules without Pydantic (Lab 22), plain text out (StrOutputParser recipes).

3 — Wiring diagram

class Joke(BaseModel)  — setup, punchline fields
       → JsonOutputParser(pydantic_object=Joke)
       → get_format_instructions()  (auto from class)
       → PromptTemplate(..., partial_variables={"format_instructions": ...})
       → WatsonxLLM
       → joke_parser
       → dict printed  {"setup": "...", "punchline": "..."}

4 — Script

from pydantic import BaseModel, Field
from langchain_core.output_parsers import JsonOutputParser
from langchain_core.prompts import PromptTemplate
from watson_llm import make_watsonx_llm

class Joke(BaseModel):
    setup: str = Field(description="question to set up a joke")
    punchline: str = Field(description="answer to resolve the joke")

joke_parser = JsonOutputParser(pydantic_object=Joke)
format_instructions = joke_parser.get_format_instructions()

joke_prompt = PromptTemplate(
    template="Answer the user query.\n{format_instructions}\n{query}\n",
    input_variables=["query"],
    partial_variables={"format_instructions": format_instructions},
)

chain = joke_prompt | make_watsonx_llm() | joke_parser
result = chain.invoke({"query": "Tell me a joke."})
print(result)  # dict with setup, punchline

6 — Variants

  • Comma list — Same pipe pattern, returns list not dict → comma-list recipe (Lab 21 Part B).
  • Hand-written JSON shape — No Pydantic class; you write format_instructions → movie-json recipe (Lab 22).
  • Stricter output — Lower TEMPERATURE via make_watsonx_llm({GenParams.TEMPERATURE: 0.1}) to reduce JSON drift.
  • Different schema — Swap Joke for any BaseModel with Field descriptions; parser regenerates instructions automatically.

Traps

  • langchain_core.pydantic_v1 in playground — Coursera notebooks use it; your venv needs from pydantic import BaseModel, Field.
  • Instructions not in prompt — {format_instructions} must appear in template AND be set via partial_variables.
  • Calling parser.parse() manually — The pipe handles it; manual parse skips prompt enforcement.
  • Expecting a Pydantic object back — invoke returns a plain dict, not a Joke instance.

Rebuild from memory

Close this page. Write the script from these steps only:

  1. Import BaseModel, Field from pydantic; JsonOutputParser, PromptTemplate, make_watsonx_llm.
  2. Define class Joke(BaseModel) with setup and punchline Fields.
  3. joke_parser = JsonOutputParser(pydantic_object=Joke).
  4. PromptTemplate with {format_instructions} + {query}; bake instructions in partial_variables.
  5. Wire prompt | make_watsonx_llm() | joke_parser.
  6. chain.invoke({"query": "Tell me a joke."}) — expect dict.
