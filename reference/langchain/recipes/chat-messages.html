Chat messages invoke

Lab 17 — SystemMessage + HumanMessage + optional AIMessage list → make_watsonx_llm().invoke([…]).

1 — Problem

A chatbot needs role-tagged turns: system instructions, user questions, and optional fake prior assistant replies. Passing one bare string cannot express “you are a book recommender” separately from the user’s question.

Not this recipe: persistent memory (Lab 33), ChatPromptTemplate pipes (Lab 20), or plain string invoke (Lab 16).

4 — Script

from watson_llm import GenParams, make_watsonx_llm
from langchain_core.messages import HumanMessage, SystemMessage, AIMessage

llama_llm = make_watsonx_llm({GenParams.MAX_NEW_TOKENS: 80})

# Example 1 — system + human
msg = llama_llm.invoke([
    SystemMessage(content="You are a helpful AI bot that assists a user in choosing the perfect book to read in one short sentence"),
    HumanMessage(content="I enjoy mystery novels, what should I read?"),
])
print(msg)

# Example 2 — include AIMessage (fake prior turn)
msg = llama_llm.invoke([
    SystemMessage(content="You are a supportive AI bot that suggests fitness activities to a user in one short sentence"),
    HumanMessage(content="I like high-intensity workouts, what should I do?"),
    AIMessage(content="You should try a CrossFit class"),
    HumanMessage(content="How often should I attend?"),
])
print(msg)

# Example 3 — human only
print(llama_llm.invoke([
    HumanMessage(content="What month follows June?"),
]))

5 — Critical lines

LineWhy it mattersIf you change it wrong…
invoke([...]) listChat API expects ordered turnsSingle string loses role separation
SystemMessage firstSets behavior before user textModel may ignore instructions if buried later
AIMessage mid-listSimulates prior assistant replyNot real memory — you fabricate the turn
MAX_NEW_TOKENS: 80Keeps demo output shortOmit → longer rambling answers

6 — Variants

  • Human only — Single HumanMessage when no system persona needed (Example 3).
  • Multi-turn fake history — Alternate HumanMessage / AIMessage before the final user question (Example 2).
  • Template + pipe — Roles in ChatPromptTemplate → chat template recipe (Lab 20).

7 — Debug

SymptomLikely causeFix
Model ignores system promptSystem message missing or lastPut SystemMessage first in list
Verbose answersDefault token limit highPass GenParams.MAX_NEW_TOKENS in factory
“Forgets” prior questionExpected automatic memoryInclude history in the list or use Lab 33 memory
Validation error on invokeWrong message typeUse langchain_core.messages classes

Traps

  • Thinking this is memory — Each invoke is stateless unless you pass prior turns in the list.
  • AIMessage from the model — Example 2 uses a hand-written fake reply to set context, not a stored prior call.
  • Mixing string and list invoke — Same llm accepts both, but list invoke is the chat-message pattern.
