Plug in WatsonxLLM for LangChain

Lab 16 — make_watsonx_llm() builds a Runnable from env. One-shot invoke or full LCEL pipe.

1 — Problem

Lab 15 proved IBM SDK works. Now you need an llm object LangChain recognizes — for invoke("…"), prompt | llm, and every recipe from Lab 19 onward — without repeating credentials in each script.

Not this recipe: raw model.generate (Lab 15), message lists (Lab 17), or template wiring (summarize).

3 — Wiring diagram

WATSONX_* env

4 — Script

import os
from watson_llm import make_watsonx_llm

print("Model:", os.environ["WATSONX_MODEL_ID"])

llama_llm = make_watsonx_llm()  # WatsonxLLM — name kept like the notebook
print(llama_llm.invoke("Who is man's best friend?"))

Playground: 16.watson_llm.invoke_test.py

llama_llm is still your WATSONX_MODEL_ID — not Coursera’s Llama unless you set that model in env.

6 — Variants

  • Plug into a pipe — prompt | make_watsonx_llm() | StrOutputParser() → summarize recipe.
  • Param override — make_watsonx_llm({GenParams.TEMPERATURE: 0.8}) → temperature compare (Lab 18).
  • Message list input — llm.invoke([SystemMessage(…), HumanMessage(…)]) → chat messages (Lab 17).

Traps

  • make_llm() — Renamed to make_watsonx_llm() in playground; old name raises NameError.
  • WatsonxLLM(model=model_inference) — Does not work in langchain_ibm 1.x. Factory passes model_id and env directly.
  • Confusing llama_llm with Llama — Variable name is notebook legacy; model comes from WATSONX_MODEL_ID.

Rebuild from memory

Close this page. Write the script from these steps only:

  1. Import make_watsonx_llm from watson_llm.
  2. Build llama_llm = make_watsonx_llm() after env is set.
  3. Call llama_llm.invoke("Who is man's best friend?").
  4. Print the result.
  5. Confirm type is WatsonxLLM, not ModelInference.
