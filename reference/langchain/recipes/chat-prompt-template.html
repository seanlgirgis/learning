ChatPromptTemplate + MessagesPlaceholder

Lab 20 — system/user tuples with {topic}, plus a msgs slot for prior turns. Format with prompt.invoke, then prompt | llm.

1 — Problem

String templates flatten everything into one blob. Chat apps need separate system and user roles, and sometimes a dynamic list of prior messages. ChatPromptTemplate builds the message list; MessagesPlaceholder injects history at invoke time.

Not this recipe: manual message lists only (Lab 17), single-string PromptTemplate (Lab 19), or output parsers (Lab 21).

3 — Wiring diagram

{"topic": "cats"}  OR  {"msgs": [HumanMessage(…)]}
       → ChatPromptTemplate.from_messages([…])
       → make_watsonx_llm({MAX_NEW_TOKENS: 120})
       → prompt.invoke(input)   # preview
       → (prompt | llm).invoke(input)   # model reply

4 — Script

from langchain_core.messages import HumanMessage
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from watson_llm import GenParams, make_watsonx_llm

llama_llm = make_watsonx_llm({GenParams.MAX_NEW_TOKENS: 120})

# --- ChatPromptTemplate (system + user with {topic}) ---
joke_prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant. Do not expand .. Just one response only."),
    ("user", "Tell me a joke about {topic}"),
])
joke_input = {"topic": "cats"}

print(joke_prompt.invoke(joke_input))
print((joke_prompt | llama_llm).invoke(joke_input))

# --- MessagesPlaceholder (slot a list of messages in) ---
history_prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant. You answer only for the request given .. Do not improvise or add anything extra. Just answer the question based on the user request."),
    MessagesPlaceholder("msgs"),
])
history_input = {"msgs": [HumanMessage(content="What is the day after Tuesday?")]}

print(history_prompt.invoke(history_input))
print((history_prompt | llama_llm).invoke(history_input))

6 — Variants

  • Format only — joke_prompt.invoke(joke_input) without LLM (playground section 1).
  • Pipe to model — joke_chain = joke_prompt | llama_llm (playground section 1b).
  • Manual messages — Build list by hand → chat messages (Lab 17).
  • String template — Single blob prompt → Lab 19.

Traps

  • Confusing prompt.invoke with model output — First call shows formatted messages; pipe call hits the LLM.
  • MessagesPlaceholder without a list — Value must be [HumanMessage(…), …], not a bare string.
  • Using string PromptTemplate for roles — System and user blur together; use ChatPromptTemplate.
