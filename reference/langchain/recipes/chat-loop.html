Interactive chat loop

Role + tone + fresh question each turn. Python while / input() wraps one invoke per line — no memory between turns.

1 — Problem

You want a REPL-style bot: read a user line, run the LCEL chain, print the answer, repeat until quit. Persona (role) and style (tone) stay constant; only question changes each iteration.

Not this recipe: conversation history across turns (Module 10), ChatPromptTemplate message lists (chat-messages), or streaming tokens.

3 — Wiring diagram

while True:
  query = input("Question: ")
  if quit → break
  {"role": role, "question": query, "tone": tone}
       → PromptTemplate("You are an expert {role}. … {question} … {tone}.")
       → WatsonxLLM
       → StrOutputParser()
       → print answer

4 — Script

from langchain_core.prompts import PromptTemplate
from langchain_core.output_parsers import StrOutputParser
from watson_llm import make_watsonx_llm, GenParams

llm = make_watsonx_llm({GenParams.MAX_NEW_TOKENS: 128})

role = "Prediction expert with deep knowledge in the topic asked."
tone = "engaging and immersive"

template = """
You are an expert {role}. I have this question {question}.
I would like our conversation to be {tone}.

Answer:
"""
prompt = PromptTemplate.from_template(template)
roleplay_chain = prompt | llm | StrOutputParser()

while True:
    query = input("Question: ")
    if query.lower() in ("quit", "exit", "bye"):
        print("Answer: Goodbye!")
        break
    response = roleplay_chain.invoke({
        "role": role,
        "question": query,
        "tone": tone,
    })
    print("Answer:", response)

5 — Critical lines

LineWhy it mattersIf you change it wrong…
roleplay_chain outside whileBuild pipe once, reuseRe-creating LLM every turn wastes time and creds
"question": query each turnOnly dynamic hole per iterationStale answer — same question every time
MAX_NEW_TOKENS: 128Short replies in interactive loopLong rambling answers; slow turns
quit / exit / bye checkClean exit without API callQuit word sent to model as a real question

6 — Variants

  • Chat message objects — Swap PromptTemplate for ChatPromptTemplate → chat-prompt-template.
  • Memory across turns — Append prior Q/A to template or use memory module (later labs).
  • Single-shot Q&A — Drop loop; use {question} + {content} → qa-context (Lab 10).

Traps

  • Expecting continuity — Each invoke is stateless. Turn 2 does not see turn 1 unless you add history yourself.
  • Rebuilding chain inside loop — Move prompt | llm | StrOutputParser() above while True.
  • Template role wins over one-word questions — Heavy persona text can swamp a short user line; simplify template if needed.

Rebuild from memory

Close this page. Write the script from these steps only:

  1. Import PromptTemplate, StrOutputParser, make_watsonx_llm, GenParams.
  2. llm = make_watsonx_llm({GenParams.MAX_NEW_TOKENS: 128}).
  3. Set constant role and tone; template with three holes.
  4. Build roleplay_chain = prompt | llm | StrOutputParser() once.
  5. while True: input → quit check → invoke with query as question → print.
