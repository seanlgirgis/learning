Recipe

Interactive chat loop

Role + tone baked into template; one invoke per user line. Not memory — each turn is fresh.

Problem

REPL-style bot: read question, run chain, print answer, repeat until quit.

Ingredients

01 02 03 04 + Python while / input()

Setup

from langchain_core.prompts import PromptTemplate
from langchain_core.output_parsers import StrOutputParser
from watson_llm import make_watsonx_llm, GenParams

llm = make_watsonx_llm({GenParams.MAX_NEW_TOKENS: 128})

Lower MAX_NEW_TOKENS in chat loops so replies finish quickly.

Recipe code

role = "Prediction expert with deep knowledge in the topic asked."
tone = "engaging and immersive"

template = """
You are an expert {role}. I have this question {question}.
I would like our conversation to be {tone}.

Answer:
"""
prompt = PromptTemplate.from_template(template)
roleplay_chain = prompt | llm | StrOutputParser()

while True:
    query = input("Question: ")
    if query.lower() in ("quit", "exit", "bye"):
        print("Answer: Goodbye!")
        break
    response = roleplay_chain.invoke({
        "role": role,
        "question": query,
        "tone": tone,
    })
    print("Answer:", response)

invoke keys (each turn)

KeyValue
rolePersona string (often constant)
questionCurrent user line from input()
toneStyle constraint (often constant)
Trap: Template role/tone wins over a one-word user question. This loop has no memory — for continuity you need a memory module later.
