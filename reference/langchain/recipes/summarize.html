Summarize in one sentence

Long text in → short summary out. Canonical LCEL recipe — every other 01–04 recipe is this shape with different template text.

1 — Problem

Analyst pastes a long report; you need a one-line summary for a ticket title. Re-typing the prompt per document wastes time and drifts wording. You need one reusable pipe: variable slot for the passage, fixed instruction sentence.

Not this recipe: multi-document RAG (Labs 28–31), chat history (Module 10), structured JSON out (Module 11).

4 — Script

from langchain_core.prompts import PromptTemplate
from langchain_core.output_parsers import StrOutputParser
from watson_llm import make_watsonx_llm

llm = make_watsonx_llm()

passage = """
The rapid advancement of technology in the 21st century has transformed
healthcare, education, and transportation. AI, ML, and IoT are changing
how we solve everyday and complex problems.
"""

template = "Summarize the {content} in one sentence."
prompt = PromptTemplate.from_template(template)
summarize_chain = prompt | llm | StrOutputParser()

print(summarize_chain.invoke({"content": passage}))

Traps

  • prompt.format(content=passage) only — Formats text but never calls the model. Need chain.invoke.
  • Passing passage as positional arg — invoke always takes a dict for PromptTemplate-first chains.
  • Duplicating this for RAG — Retrieval adds retriever + context hole; use Lab 31 recipe when built, not this page.

Rebuild from memory

Close this page. Write the script from these steps only:

  1. Import PromptTemplate, StrOutputParser, make_watsonx_llm.
  2. Build llm = make_watsonx_llm() after env is set.
  3. PromptTemplate.from_template with exactly one hole: {content}.
  4. Wire prompt | llm | StrOutputParser().
  5. chain.invoke({"content": passage}) and print.
