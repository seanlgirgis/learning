Classify into fixed list

Text + closed category list in → single label out. Two template holes: {text} and {categories}.

1 — Problem

Support ticket or article snippet arrives; you must bucket it into one label from a list you control (e.g. Music, Technology). Open-ended tagging drifts — you need a closed set spelled out in the prompt.

Not this recipe: multi-label extraction with JSON parsers (Module 11), sentiment with free-form bullet sections (review-lambda), or embedding-based classifiers.

3 — Wiring diagram

{"text": passage, "categories": "A, B, C."}
       → PromptTemplate("Classify the {text} into one of the {categories}. …")
       → WatsonxLLM
       → StrOutputParser()
       → str label printed

4 — Script

from langchain_core.prompts import PromptTemplate
from langchain_core.output_parsers import StrOutputParser
from watson_llm import make_watsonx_llm

llm = make_watsonx_llm()

text = """
The concert last night was an exhilarating experience with outstanding
performances by all artists.
"""

categories = "Entertainment, Food and Dining, Technology, Literature, Music."

template = """
Classify the {text} into one of the {categories}.

Category:
"""
prompt = PromptTemplate.from_template(template)
classification_chain = prompt | llm | StrOutputParser()

print(classification_chain.invoke({"text": text, "categories": categories}))

Playground: 11.lcel.classification.py

6 — Variants

  • Stricter labels — Lower TEMPERATURE via make_watsonx_llm({GenParams.TEMPERATURE: 0.1}) → temperature-compare.
  • Structured JSON out — Swap StrOutputParser for a JSON parser → movie-json (Module 11).
  • Q&A shape — Replace classify instruction with question + context holes → qa-context (Lab 10).

Traps

  • Categories buried in code only — If {categories} is empty or generic, the model free-forms labels.
  • Expecting multi-label — Template says one of; for many tags you need different prompt + parser.
  • Using {content} hole name — Lab uses text; mismatched key → KeyError.

Rebuild from memory

Close this page. Write the script from these steps only:

  1. Import PromptTemplate, StrOutputParser, make_watsonx_llm.
  2. Build llm = make_watsonx_llm() after env is set.
  3. Define text and categories strings.
  4. Template: Classify the {text} into one of the {categories}. + Category:.
  5. Wire prompt | llm | StrOutputParser(); invoke with both keys; print.
