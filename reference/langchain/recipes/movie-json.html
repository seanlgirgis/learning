Movie JSON parser

Bare JsonOutputParser() + hand-written format_instructions — no Pydantic class. You define the JSON shape in the prompt yourself.

1 — Problem

A movie lookup tool needs title, director, year, and genre as separate fields from the LLM. Lab 21 used a Pydantic class to auto-generate format rules; Lab 22 drops the class and makes you write the JSON contract explicitly — closer to real prompts where schema lives in the template, not Python types.

Not this recipe: Pydantic-driven JSON (Lab 21 Part A), comma lists (Lab 21 Part B), plain text summarize (StrOutputParser).

4 — Script

from langchain_core.output_parsers import JsonOutputParser
from langchain_core.prompts import PromptTemplate
from watson_llm import make_watsonx_llm

json_parser = JsonOutputParser()

format_instructions = """RESPONSE FORMAT: Return ONLY a single JSON object—no markdown, no examples, no extra keys.  It must look exactly like:
{
  "title": "movie title",
  "director": "director name",
  "year": 2000,
  "genre": "movie genre"
}

IMPORTANT: Your response must be *only* that JSON.  Do NOT include any illustrative or example JSON."""

prompt_template = PromptTemplate(
    template="""You are a JSON-only assistant.

Task: Generate info about the movie "{movie_name}" in JSON format.

{format_instructions}
""",
    input_variables=["movie_name"],
    partial_variables={"format_instructions": format_instructions},
)

movie_chain = prompt_template | make_watsonx_llm() | json_parser
result = movie_chain.invoke({"movie_name": "The Matrix"})

print(f"Title: {result['title']}")
print(f"Director: {result['director']}")
print(f"Year: {result['year']}")
print(f"Genre: {result['genre']}")

6 — Variants

  • Pydantic auto-schema — Class drives instructions → JSON Joke recipe (Lab 21 Part A).
  • More fields — Add "rating", "cast" to the example JSON in format_instructions.
  • Stricter output — Lower TEMPERATURE via make_watsonx_llm({GenParams.TEMPERATURE: 0.1}).
  • Different entity — Swap movie for book, album — same bare-parser pattern, new format string.

Traps

  • Calling get_format_instructions() on bare parser — Works but gives generic rules; Lab 22 wants you to hand-write the movie-specific contract.
  • Adding a Pydantic class here — Valid pattern, but that's Lab 21 Part A, not this exercise.
  • Model returns example placeholder values — "movie title" instead of real title — prompt must say generate real data for {movie_name}.
  • Markdown fences — Even with instructions, some models wrap JSON; tighten prompt or post-strip fences before parse.

Rebuild from memory

Close this page. Write the script from these steps only:

  1. Import JsonOutputParser, PromptTemplate, make_watsonx_llm — no Pydantic.
  2. json_parser = JsonOutputParser() — bare, no pydantic_object.
  3. Write format_instructions string with example JSON keys: title, director, year, genre.
  4. PromptTemplate with {movie_name} + {format_instructions}; bake instructions in partial_variables.
  5. Wire prompt | make_watsonx_llm() | json_parser.
  6. chain.invoke({"movie_name": "The Matrix"}) — access result['title'] etc.
