CommaSeparatedListOutputParser

Model returns comma-separated text → parser splits into a Python list. Same format-instructions + pipe pattern as JSON, different return type.

1 — Problem

You ask the model for a set of items — five programming languages, three cloud providers — and need them as a Python list for iteration, dropdowns, or batch processing. Raw model text might use bullets, numbering, or newlines instead of commas; the parser normalizes to ["Python", "Java", ...].

Not this recipe: structured JSON with named fields (Lab 21 Part A), hand-written movie JSON (Lab 22), single plain-text answer (StrOutputParser).

4 — Script

from langchain_classic.output_parsers import CommaSeparatedListOutputParser
from langchain_core.prompts import PromptTemplate
from watson_llm import make_watsonx_llm

list_parser = CommaSeparatedListOutputParser()
format_instructions = list_parser.get_format_instructions()

prompt = PromptTemplate(
    template="Answer the user query.\n{format_instructions}\nList five {subject}.",
    input_variables=["subject"],
    partial_variables={"format_instructions": format_instructions},
)

chain = prompt | make_watsonx_llm() | list_parser
print(chain.invoke({"subject": "programming languages"}))

6 — Variants

  • JSON object — Named fields → dict via JSON Joke recipe (Lab 21 Part A).
  • Different count — Change "List five" to "List three" or "List ten" in template text.
  • Different subject hole — Rename {subject} to {topic} — keep invoke key aligned.
  • Stricter output — Lower TEMPERATURE to reduce creative formatting (bullets instead of commas).

Traps

  • Expecting dict back — This parser returns list. JSON joke recipe returns dict.
  • Import from langchain_core — CommaSeparatedListOutputParser lives in langchain_classic.output_parsers in your venv.
  • Model uses bullets or numbers — Without format_instructions in the prompt, output won't comma-split cleanly.
  • Commas inside item names — "Python, the language" splits wrong — prompt for simple one-word items or use JSON parser instead.
