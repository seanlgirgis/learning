Q&A from context

Question + passage in → grounded answer out. Same LCEL pipe as Summarize, but two template holes: {question} and {content}.

1 — Problem

User asks a question about a document you already have. The model must answer from the supplied passage — not from general knowledge. If the text does not support an answer, the prompt tells the model to say Unsure about answer.

Not this recipe: retrieval over a vector store (Labs 28–31), multi-turn chat memory (Module 10), or open-ended trivia with no context block.

4 — Script

from langchain_core.prompts import PromptTemplate
from langchain_core.output_parsers import StrOutputParser
from watson_llm import make_watsonx_llm

llm = make_watsonx_llm()

content = """
The solar system consists of the Sun, eight planets, their moons, dwarf
planets, and smaller objects. The inner planets—Mercury, Venus, Earth,
and Mars—are rocky and solid. The outer planets are larger and gaseous.
"""

question = "Which planets in the solar system are rocky and solid?"

template = """
Answer the {question} based on the {content}.
Respond "Unsure about answer" if not sure about the answer.

Answer:
"""
prompt = PromptTemplate.from_template(template)
qa_chain = prompt | llm | StrOutputParser()

print(qa_chain.invoke({"question": question, "content": content}))

Playground: 10.lcelanswer.py (lab also shows RunnableLambda(format_prompt) — see review-lambda).

6 — Variants

  • One hole only — Drop {question}, keep {content} → summarize recipe (Lab 09).
  • Lambda formatter — RunnableLambda(format_prompt) | llm → review-lambda (Lab 14 pattern; Lab 10 uses the same trick).
  • Stricter grounding — Add Answer only from the passage. Do not use outside knowledge. to the template.

Traps

  • Only one invoke key — Two holes means two dict keys. invoke({"question": q}) alone fails.
  • Swapping hole names — {content} in template must be "content" in dict, not "context" or "passage".
  • Treating this as RAG — You supply the passage manually. Retrieval + inject is a different lab chain.
