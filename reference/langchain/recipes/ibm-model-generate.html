Test watsonx with model.generate

Lab 15 — IBM SDK smoke test via watson_helper. Dict response, no LangChain pipe.

1 — Problem

Before wiring LCEL chains, you need proof that WATSONX_* env vars load and IBM ModelInference returns text. This is the notebook LOCAL SETUP check — one string in, one completion out.

Not this recipe: LangChain pipes (Lab 16), chat roles (Lab 17), or template holes (Lab 19).

4 — Script

from watson_helper import credentials, model, model_id

print("Model:", model_id)
print("URL:", credentials["url"])

prompt = "In today's sales meeting, we "
msg = model.generate(prompt)
print(msg["results"][0]["generated_text"])

6 — Variants

  • LangChain path — Swap to make_watsonx_llm() → LLM plug-in recipe (Lab 16).
  • Param tuning — Override generation params when building the model in watson_helper.py, or use temperature compare with two make_watsonx_llm() instances.
  • Closed prompt — Use a full question instead of a half-sentence if completions look like grammar drills.

Traps

  • Putting model in a pipe — ModelInference is not a LangChain Runnable. Use Lab 16.
  • Expecting a string from generate — You must index into results[0]["generated_text"].
  • Skipping this smoke test — If Lab 15 fails, fix env/credentials before Labs 16–20.
