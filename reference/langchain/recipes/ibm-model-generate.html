Recipe · Module 06

Test watsonx with model.generate

Lab 15 pattern — IBM SDK, dict response. Not for LangChain pipes.

Problem

Confirm WATSONX_* env is loaded and watsonx returns text — same as the notebook LOCAL SETUP demo.

Ingredients

06 ModelInference 07 watson_helper

Playground lab (15.watson_helper.generate_test.py)

# Run set_env.ps1 before this script
from watson_helper import credentials, model, model_id

print("Model:", model_id)
print("URL:", credentials["url"])

prompt = "In today's sales meeting, we "
msg = model.generate(prompt)
print(msg["results"][0]["generated_text"])

Jupyter notebook equivalent

from coursera_watsonx_model import credentials, model, model_id

print("Model:", model_id)
print("URL:", credentials["url"])

msg = model.generate("In today's sales meeting, we ")
print(msg["results"][0]["generated_text"])

Reading the response

ExpressionWhat you get
msgFull IBM JSON as a Python dict
msg["results"][0]First completion object
["generated_text"]The actual model text (string)
Open prompt trap: A half-sentence like "In today's sales meeting, we " may produce odd completions (e.g. grammar drills). That is model behavior, not a broken helper.
Next step: For LangChain, switch to LLM plug-in recipe — do not put model in a pipe.
