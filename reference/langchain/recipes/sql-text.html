Generate SQL text

English spec in → SQL string out. One hole: {description}. No database connection.

1 — Problem

Analyst describes tables, columns, and filters in plain English; you need a query string to paste into a SQL client. The chain returns text only — it does not execute against a live database.

Not this recipe: SQL agents with tool calls, schema introspection from a connected DB, or validating results row-by-row.

3 — Wiring diagram

{"description": english_spec}
       → PromptTemplate("Generate an SQL query only … based on the {description}")
       → WatsonxLLM
       → StrOutputParser()
       → SQL str printed

4 — Script

from langchain_core.prompts import PromptTemplate
from langchain_core.output_parsers import StrOutputParser
from watson_llm import make_watsonx_llm

llm = make_watsonx_llm()

description = """
Retrieve the names and email addresses of all customers from the
'customers' table who have made a purchase in the last 30 days.
The table 'purchases' contains a column 'purchase_date'.
"""

template = """
Generate an SQL query only with no explanation or commentary based on the {description}

SQL Query:
"""
prompt = PromptTemplate.from_template(template)
sql_generation_chain = prompt | llm | StrOutputParser()

print(sql_generation_chain.invoke({"description": description}))

Playground: 12.lcel.generatesql.py (lab stores the spec in a variable named story but invokes with {"description": story}).

6 — Variants

  • Summarize shape — One hole, different instruction → summarize (Lab 09).
  • Lower creativity — make_watsonx_llm({GenParams.TEMPERATURE: 0.1}) for stabler syntax.
  • RunnableLambda formatter — Pre-process description before LLM → review-lambda (Lab 14).

7 — Debug

Traps

  • Variable name ≠ invoke key — Lab calls the string story; template hole is still {description}.
  • Expecting execution — This chain prints text. Connecting to SQLite/Postgres is a different pattern.
  • Agent confusion — Fixed prompt | llm is not an agent that looks up Italy’s population from a live DB.

Rebuild from memory

Close this page. Write the script from these steps only:

  1. Import PromptTemplate, StrOutputParser, make_watsonx_llm.
  2. Build llm = make_watsonx_llm() after env is set.
  3. English spec in description with tables, columns, filters.
  4. Template: generate SQL only from {description}; end with SQL Query:.
  5. Wire prompt | llm | StrOutputParser(); invoke({"description": description}); print.
